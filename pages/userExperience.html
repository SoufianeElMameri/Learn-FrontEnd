User Experience

  1. What is a User Experience?

  2. User Experience is exactly what it sounds like. When a user comes on a website, they have an experience. If the site is well designed and caters for the user's needs, then they have a good experience. Otherwise, they have a bad experience and are unlikely to return to the website.
  3. This interfers with how a user interacts with a webpage and how easy it may be for them.

User Experience Proccess

  • When designing a website (or any product really), the designer must answer a number of questions to decide how to best proceed:
    • What is the purpose of the site?
    • Who will use the site?
    • What is the main reason a user will come to the site?
    • What task is the user looking to accomplish?
    • How often will the user be on the site?
    • How long will the user spend on the site each visit?
    • Is the site for business or recreation?
    • Any specific goals for the site?

What Affects a User's Experience?

A great user experience depends on many factors. Depending on what type of site you are building, these could be very different, but there are a few that will come into play on all sites:

Usability
Put simply, this is ease of use. If the user finds your site difficult or confusing, they will find another site to use. Your job when designing a site is to let the user accomplish their task with as little thought and few clicks as possible. As mentioned in Layout and Navigation, if you design with the user in mind, then everything sould be clear and easy for them to understand.
Layout
Designing the layout of the page with the user in mind is essential. If the right things are given priority and are made obvious, it is much easier for the user to see what to do when they arrive on the page. The smaller the learning curve, the better.
Navigation
Finding their way around the site should be straightforward for the user. If they have to guess what to press or where to go then you have not designed your navigation well.
Colour
Color can be used to great effect for good user experience. If done right, it can bring out all sorts of reactions and emotions from a user.

Dos & Don'ts of User Experience

Do

  • Keep language simple & concise
  • Make things clear to the user
  • Highlight the important functions
  • Provide prompt & error messages when needed
  • Use familiar interfaces that the user is already used to
  • Provide as pleasant and easy an experience as possible

Don't

  • Make things difficult for the user
  • Hide any features unless necessary
  • Use text when an image or button will do the job better & quicker
  • Overwhelm the user with everything all at once
  • Use colours that clash and make it hard for the user to look at the site
  • Use new icons & button labels for features that already have a well known label

Making a User's Experience Enjoyable

What are the qualities that create a great user experience?

  • Functional user interface & layout
  • Clear feedback for the user when attempting tasks
  • Generate positive feelings of success, competency, and mastery among users
  • Users have a clear mental model of the interface
  • Users can confidently predict what will happen in response to their actions
  • Users can forget the interface and concentrate on the task
  • Calming effect - colour use, positive feedback
